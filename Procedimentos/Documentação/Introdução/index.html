    Introdução

    Objetivo

    • Facilitar a troca de informação entre os setores;
    • Padronização;
    • Correção de erros;
    • Aprendizado contínuo.

    Pré requisitos para elaboração da documentação

    • Conhecimento básico em markdown
    • Instalação do python
    • Instalação do MKDocs

    Markdown

    Os arquivos da documentação são escritos em markdown.
    Markdown é uma linguagem de marcação que utiliza simbolos para formação, os principais simbolos podem ser consultados no cheat sheet.

    MKDocs

    Essa documentação é escrita com markdown e gerada pelo MKDocs, uma ferramenta de feração de sites estáticos.

    O MKDocs transforma os arquivos escritos em Markdown em páginas web.

    Para melhor apresentação da documentação o tema escolhido foi o Material.

    Pré requisitos

    Para utilização do MKDocs é necessário ter o python e o gerenciador de pacotes pip instalados.
    O python pode ser baixado na página de downloads.

    Instalação do MKDocs

    No terminal / cmd digite:

    pip install mkdocs

    Para instruções mais detalhadas e solução de dúvidas, vá ao site de instalação do MKDocs

    Utilização do MKDocs

    Criação da documentação

    Para rodar o https://www.mkdocs.org/, no terminal / cmd rode o comando:

    mkdocs new nome_da_pasta_de_documentação

    Para esse projeto foi utilizada a pasta padrão do https://www.mkdocs.org/.

    mkdocs new .

    Após rodar o comando, o https://www.mkdocs.org/ irá criar uma pasta .docs, onde serão armazenadas todos os arquivos da documentação.

    Vizualização da documentação

    Para vizualizar a documentação criada, rode o comando:

    mkdocs serve

    O comando irá rodar um servidor local (na máquina do usuário) onde a documentação pode ser vizualizada.

    Gerar a documentação

    Para que a documentação seja acessível para todos os usuários ela deve ser independete, ou seja, os usuários que irão vizualizar a documentação não devem se preocupar com os pré requisitos.
    Para isso, após o término da documentação rode o comando:

    mkdocs build

    O comando transformará todos os arquivos https://www.markdownguide.org/ em arquivos html que podem ser acessados por todos os navegadores, esses arquivos serão criados na pasta site.

    Material

    O material é um tema aplicado ao MKDocs para alterar o layout das páginas e adicionar algumas funcionalidades. O material é o tema mais utilizado nas documentações atuais.

    Instalação do Material

    No terminal / cmd digite:

    pip install mkdocs-material

    Plugins

    MKDocs macros

    O MKDcos macros permite a criação de variáveis no arquivo mkdocs.yml do MKDocs para facilitar futuras manutenções.

    Instalação

    No terminal / cmd digite:

    pip install mkdocs-macros-plugin

    Utilização

    extra:
    nome_grupo:
        nome_variavel: valor_variavel

    extra:
    nomes:
        perfil_viga: perfil U

    Referência:

    O {{ no such element: dict object['perfil_viga'] }}

    Resultado:

    O perfil U...

    Disponibilar a documentação online

    Todos os arquivos referentes a documentação estão disponíveis no repositório documentacao_gimi do github.

    Após qualquer atualização ou ao fim do dia de trabalho é recomendado o upload da documentação no repositório do github.

    O repositório do projeto está vinculado com o netlify, um site de hospedagem de sites estáticos, onde a documentação pode ser acessada por qualquer dispositivo.
